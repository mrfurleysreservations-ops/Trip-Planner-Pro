Gear Lives on the Packing Tab

The two-tab split: Packing is "what I'm physically transporting" (clothes + outfits + my gear). Supplies is "what the group is buying for this trip." No overlap.

The clean line

The question I use to decide where something belongs: "If this trip gets cancelled, does the item still exist and have value to me?" Yes → Gear (Packing tab). No → Supply (Supplies tab).

Gear is a persistent library that lives in your profile, built up across trips. When you're planning a specific trip, you pull from the library to say "this tent and those sleeping pads are coming on this trip." Other trip members see a group roll-up so nobody double-packs and the tent doesn't get forgotten.

Supplies stays trip-scoped: things bought or sourced for this specific weekend that get consumed or thrown away. Firewood, propane, ice, bachelor-party sashes. It doesn't care about your gear library.

Packing → Gear

  • 4-person tent
  • Coleman 2-burner stove
  • Sleeping pads / bags
  • Camp chairs
  • Cookware set, lanterns, hatchet
  • First aid kit

Supplies

  • Firewood, propane, charcoal
  • Ice, batteries, duct tape
  • Paper plates, trash bags
  • Sunscreen, bug spray
  • Party decorations, sashes, games

Power users (your preference): build the gear library once in your profile, then assemble trips from it. Casual users: skip the library and just add items to Supplies per-trip. The Gear pill on Packing is opt-in — if you haven't built a library, it shows a "Start your gear library" empty state. One doesn't gate the other.

What's on screen

  • New "Gear" pill joins the existing view switcher alongside Group / Outfits / Pack & Go ✓.
  • Person tabs scope the view — Joe's gear here, tap Sarah to see hers. "Everyone" tab shows the full aggregate.
  • Your Gear section is interactive — check off as you load the car. Stored as a per-user, per-trip packed state (same pattern as existing packing_items).
  • Others are bringing is a read-only roll-up. Gives you air-cover so the tent doesn't get forgotten.
  • Pull from library opens the next frame — picks from your persistent gear library and adds to this trip.

How the library pull works

  • Source of truth is the library (`gear_items` in user profile). Adding to a trip creates a join row (`trip_gear`) — the library item itself isn't duplicated.
  • "On trip" chip shows what's already pulled in, so you can see state at a glance when reopening.
  • Search + section grouping — library will grow; search is the escape valve.
  • "Don't have it yet?" link goes to Profile → My Gear to add new library items without leaving the flow.
  • Removing from trip ≠ deleting from library — uncheck just removes the `trip_gear` join.
