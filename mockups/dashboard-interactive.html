🔔 Notification bubble logic

Chats bubble: show when count of unread messages across all trips the user is a member of is greater than 0. Use trip_chat_messages created after last-read timestamp per trip. Count = total unread across all trips.
Friends bubble: show when count of pending friend requests targeting the current user is greater than 0. Query friend_links where target_user_id = current user and status = 'pending'.
Alerts bubble: not currently mocked — same pattern can apply (unread alert count). Tell me if you want this added.
Bubble caps at "9+" when count exceeds 9 for visual consistency.

🎯 Pattern rationale

Trip sub-nav lives at the bottom of each trip page. The dashboard's top-level nav lives at the top so users can clearly distinguish "where in the app" vs "where in this trip." Same visual language (icon + label, 3px accent underline indicator, 56px height, transparent background). The two nav bars never coexist — top-level nav only on /dashboard, /chats, /friends, /gear, /profile, /alerts; bottom sub-nav only inside a trip.
