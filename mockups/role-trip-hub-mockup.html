💡 How Chat fits the model

Chat was the newest addition and the biggest test of the "nothing disappears" rule. Here's how it threads through: (1) All 7 tabs including Chat render for every role. (2) Chat tab POSITION shifts per role — All In keeps it default (3rd), Just Here pulls it to 2nd (logistics pings matter), Vibes Only keeps it mid-nav but mutes notifications. (3) The unread indicator itself changes shape — red count badge for users who want it, gray dot for Vibes Only so chat isn't visually pestering them. (4) Notification defaults per role are stored on trip_members.chat_notification_level — driven by role_preference at pick time but overridable.
