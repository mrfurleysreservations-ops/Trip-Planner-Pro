Trip Tab Layout Standard — v2 (faithful to what exists today)

Every trip tab — Itinerary, Expenses, Packing, Notes, Meals, Group — follows this skeleton. Tabs keep their existing content and behavior; we only reorganize the top region so the navigation controls stay pinned while the body scrolls. Red "NEW" chip = added in this change. Everything else is preserved.
1. Sticky header
Back · page title · 0–3 action buttons (Export, Import, +, Save, etc.). Height ~40px.
2. Sticky pill (optional)
Primary view switcher. 2 or 3 options allowed (Expenses: 2 · Packing: 3 · Notes: 2 new + existing filter stays). Some tabs have no pill today — OK to omit.
3. Sticky person row (optional)
For Packing only — horizontal scroll of family-member tabs. Persists across view switches.
4. Sticky filter/search row (optional)
Horizontal chip row OR search input OR day chips. Tab-specific.
5. Scrollable body
flex: 1; overflow-y: auto, bottom padding 68px so last item clears the sub-nav.
6. Fixed sub-nav
Unchanged. 52px, 6 tabs. position: fixed; bottom: 0.
7. Optional FAB
Bottom-right at bottom: 68px, only where "+ Add one" is the primary action (tabs with an Add button already in the header don't need both).
Layering rules
All sticky rows share one container. Nothing below the sticky container is sticky. No orphan position: sticky in the body.
