List-view details

  • Attendees are read-only here — editing lives on Itinerary. We just show who's in so you know what to multiply by.
  • "per-person × 6 attending" hint in the Ingredients header reminds you the qty is per person.
  • Drag handle for reordering ingredients (stored in `sort_order`).
  • Tap a row → it expands inline into the edit box (see next frame) — no modal-in-modal.
  • Claim box at bottom highlights green when it's yours. Host sees a "Change" link to reassign.

Edit/Add details

  • Inline expansion — tapping a row or "+ Add" opens this form in place. No second modal stacked on top.
  • Four fields: name, qty-per-person, unit, grocery section. Plus optional note.
  • Unit dropdown: ea, lb, oz, g, kg, cup, tbsp, tsp, bottle, bag, pkg. (Free-text allowed — camping has weird units.)
  • Section dropdown is the fixed 9-value list — that's what drives Grocery view ordering.
  • Remove deletes the meal_item row; Grocery view recomputes next load.
  • Add ingredient uses the exact same form state, just empty.

Supply editor details

  • Same bottom-sheet pattern as the meal editor — maxW 480, 20px top radius, slideUp, handle bar.
  • Status as three buttons (not a dropdown) so the state transition feels tactile — Needed → Claimed → Purchased.
  • Claim toggle + member picker — tap "I'll bring it" for self, or pick another member. Picker is how host assigns to family (RLS owner-clause handles it).
  • Same sheet for "+ Supply" — opens empty with status=Needed and claimed_by=null.
  • Notes → Supply finalization opens this sheet prefilled (name from note title, note body → Notes field, `source_note_id` set server-side).
