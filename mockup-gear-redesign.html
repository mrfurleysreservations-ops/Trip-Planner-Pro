What's changing vs. Phase 1

  • Layout: inline expanding cards → flat row list grouped by location. No more "jumbled" expanded cards pushing other bins down the page.
  • Edit surface: inline expansion → bottom-sheet modal. Same modal on /gear and inside a trip. One edit UI to maintain.
  • New bin button: header button → floating ＋ FAB bottom-right.
  • Nested bins: new. Schema change — gear_bins.parent_bin_id (self-reference, nullable).
  • Sticky CTA: input + "Add" always pinned to the bottom of the sheet (matches Add Booking pattern).

One thing I need you to decide: how deep does nesting go?

  • One level only (top-level bins contain child bins; child bins only contain items). Simple model, easy to reason about, covers most real-world cases. Camp Kitchen → Cookware Tote → items.
  • Arbitrary depth (any bin can contain bins and items). More flexible, but UI gets harder — breadcrumbs can grow long, and on small screens the sheet starts to feel like a file browser.

The rest of the build is the same either way; only the validation logic + a couple of UX details differ.
